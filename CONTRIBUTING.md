# Contributing to AuraLens

Thank you for your interest in contributing! We welcome bug reports, feature requests, and pull requests.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue on GitHub. Include:
* Your Chrome browser version.
* A clear description of the bug.
* Steps to reproduce the issue.
* Console logs (if applicable).

### 2. Suggesting Features
Have an idea to make AuraLens better? Open an issue and use the "Feature Request" label. Explain the problem your feature solves and how you envision it working.

### 3. Pull Requests
1. Fork the repository.
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Test your changes locally by loading the unpacked extension in Chrome.
5. Commit your changes (`git commit -m 'Add amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request on the main repository.

## Coding Standards
* Keep the architecture simple and lightweight (Vanilla JS/CSS preferred).
* Ensure any visual changes maintain accessibility standards.
* Do not introduce dependencies unless absolutely necessary.
